🙇 Apa itu SSH Key Github?

SSH (Secure Shell) adalah protokol yang memungkinkan user terhubung dan mengotentikasi ke remote server dan layanan secara aman. Dalam konteks GitHub, SSH digunakan untuk menghubungkan komputer lokal dengan akun GitHub tanpa perlu memasukkan username dan password setiap kali melakukan operasi Git.

🙇 Bagaimana cara kerjaya?

🙎 Bayangkan seperti sistem kunci dan gembok:

  • Kunci privat disimpan di komputer kita
  • Kunci publik diberikan ke Github sebagai gembok

    • 🙇 Lalu, apa bedanya dengan login biasa?

    Proses Otentikasi

    Ketika ingin terhubung ke GitHub, komputer akan mengirim pesan yang "dikunci" dengan kunci privat. GitHub menerima pesan ini dan mencoba "membukanya" dengan kunci publik yang tersimpan di sana. Jika berhasil dibuka, GitHub tahu itu benar-benar dan mengizinkan koneksi.

    Keamanan

    Karena hanya kita yang memiliki kunci privat, hanya kita yang bisa membuat pesan yang bisa "dibuka" oleh kunci publik di GitHub. Ini seperti hanya pemilik kunci yang bisa membuka gembok di pintu GitHub. dengan SSH tidak perlu login akun secara manual, (email, password, konfigurasi kode melalui email), kita hanya perlu memasukan kode melalui git log, ini sangat memudahkan jika bekerja dengan komputer yang berbeda. Lebih aman daripada password, karena kunci privat tidak pernah dikirim melalui internet. Lebih nyaman, karena tidak perlu mengetik password.

    Penggunaan:

    Setiap kali melakukan operasi Git (seperti push atau pull), sistem ini bekerja di latar belakang. kita tidak perlu memasukkan password setiap kali, karena "kunci" yang melakukan otentikasi.

    🙇 Bagaimana jika kita mengganti Sistem Operasi, apakah password akan hilang?

    🙎 Tidak, Password akun GitHub disimpan di server GitHub, bukan di komputer. Jika mengganti sistem operasi, kunci SSH (baik privat maupun publik) tersimpan di komputer.dan jika kita menginstal sistem operasi baru, kunci SSH ini akan hilang karena tersimpan di hard drive yang di-format. Password akun GitHub tetap aman dan tidak terpengaruh. kita masih bisa login ke akun GitHub menggunakan username dan password seperti biasa. dan tidak bisa lagi menggunakan koneksi SSH dari komputer tersebut ke GitHub.

    🙎 Jadi jangan install2 terus lah kalau tidak terlalu urgen wkwkwkwkw

    Apa yang perlu dilakukan?
  • membuat kunci SSH baru di sistem operasi yang baru.
  • Tambahkan kunci publik baru ini ke akun GitHub.
  • Hapus kunci SSH lama dari pengaturan GitHub (karena sudah tidak bisa digunakan).

    • 🙇 Kesimpulannya?

    SSH Key adalah cara aman dan efisien untuk terhubung ke GitHub. Sekali setup, kerja jadi lebih mudah. Yang penting jaga kunci privat dan lakukan manajemen yang baik. Jangan pernah membagikan kunci privat
